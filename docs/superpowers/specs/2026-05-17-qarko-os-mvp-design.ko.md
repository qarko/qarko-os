# QARKO OS MVP 설계서

## 목적

QARKO OS는 1인 AI 회사를 위한 실시간 운영체제입니다.

사용자가 어떤 사업 아이디어를 가지고 있든, QARKO OS는 그 아이디어를 프로젝트, 목표, 워크플로, AI 역할, 로그, 승인, 산출물, 플러그인, 자동화 범위가 있는 운영 공간으로 바꿔줍니다.

QARKO OS는 특정 사업 분야에 묶이면 안 됩니다. AI 인플루언서, 유튜브 쇼츠, SEO 블로그, 제휴 마케팅, SaaS, 로컬 비즈니스 대행, 기타 어떤 아이디어든 같은 운영 구조 안에서 다룰 수 있어야 합니다.

최종 상용 목표는 Windows 데스크톱 앱입니다. 다만 첫 MVP는 웹앱으로 먼저 만들어 핵심 제품 경험을 빠르게 검증하고, 이후 Tauri로 Windows 데스크톱 앱화할 수 있는 구조로 설계합니다.

## 제품 포지션

QARKO OS는 세 가지를 결합합니다.

- 1인 사업 운영 대시보드
- Codex Windows 앱 같은 실시간 실행 인터페이스
- Hermes 기반 에이전트와 모델 워크스페이스

제품의 중심 화면은 사업 운영 대시보드입니다. Codex 같은 실행 패널과 AI 에이전트 역할은 대시보드 안에 녹아들어야 하며, 제품 전체가 단순 채팅앱처럼 보이면 안 됩니다.

사용 흐름은 다음과 같습니다.

1. 사용자가 QARKO OS를 열면 자신의 1인 사업 현황을 봅니다.
2. 사용자가 사업 프로젝트를 만들거나 기존 프로젝트를 선택합니다.
3. QARKO OS는 아이디어를 목표, 단계, 작업, 에이전트, 승인 게이트, 산출물로 정리합니다.
4. AI 에이전트는 사용자가 정한 자동화 범위 안에서 안전한 작업을 실행합니다.
5. 위험하거나 사용자가 지정한 작업은 명시적 승인을 요구합니다.
6. 로그와 산출물을 통해 AI가 무엇을 했는지 보입니다.
7. 워크플로와 플러그인은 이후 커뮤니티에서 공유 가능한 구조로 확장합니다.

## MVP 목표

MVP의 목표는 QARKO OS가 챗봇이 아니라 1인 AI 회사를 운영하는 대시보드처럼 느껴지는지 검증하는 것입니다.

MVP에 포함할 기능:

- 워크스페이스 대시보드
- 프로젝트 목록과 활성 프로젝트 화면
- 새 프로젝트 생성 흐름
- 프로젝트 시작 시 자동화/승인 범위 설정
- AI 역할 카드
- 워크플로와 작업 카드
- Codex 같은 실시간 실행 패널
- 로그 타임라인
- 승인 대기열
- 산출물 보관함
- Codex Windows 앱 플러그인 경험을 참고한 플러그인 갤러리
- Hermes/모델 연결 상태 mock 표시
- 향후 Hermes, 플러그인, 데스크톱 기능을 붙일 수 있는 adapter 구조

MVP에서 제외할 기능:

- 실제 플러그인 런타임
- 실제 커뮤니티 마켓
- 결제/구독
- 실제 게시 자동화
- 다중 사용자 협업
- 깊은 Hermes 실행 연동
- 실서비스용 인증

위 기능들은 나중에 붙을 수 있도록 화면 자리와 데이터 구조만 준비합니다.

## 첫 화면

기본 홈 화면은 워크스페이스 대시보드입니다.

기존 사용자는 다음을 봅니다.

- 현재 프로젝트
- 오늘의 우선순위
- 진행 중인 AI 작업
- 승인 요청
- 최근 산출물
- 활성 플러그인
- Hermes/모델 상태

새 사용자나 빈 워크스페이스는 새 프로젝트 생성 흐름으로 안내합니다.

이 구조는 QARKO OS가 운영 콘솔처럼 느껴지게 하면서도 온보딩을 단순하게 만듭니다.

## 화면 구조

MVP는 Codex Windows 앱을 참고한 3분할 데스크톱 스타일 레이아웃을 사용합니다.

### 왼쪽 사이드바

왼쪽 사이드바에는 다음을 둡니다.

- 워크스페이스 전환
- 프로젝트 목록
- 새 프로젝트 버튼
- 템플릿
- 플러그인
- 커뮤니티
- 설정

사이드바는 마케팅 사이트 메뉴가 아니라 운영체제의 고정 조작 패널처럼 느껴져야 합니다.

### 중앙 대시보드

중앙 영역은 워크스페이스 대시보드 또는 활성 프로젝트 화면을 보여줍니다.

워크스페이스 대시보드에는 다음을 보여줍니다.

- 사업 전체 요약
- 프로젝트 상태 카드
- 오늘의 우선순위
- 승인 대기 미리보기
- 진행 중 작업 미리보기
- 산출물 미리보기

활성 프로젝트 화면에는 다음을 보여줍니다.

- 프로젝트 목표
- 자동화 범위
- 워크플로 단계
- 작업 카드
- 배정된 AI 역할
- 주요 리스크
- 다음 추천 액션

### 오른쪽 실행 패널

오른쪽 패널은 Codex 같은 실시간 작업 화면입니다.

포함 요소:

- 현재 실행 제목
- 활성 AI 역할
- 선택된 모델 또는 mock 모델
- 단계별 작업 진행
- 스트리밍처럼 보이는 로그
- 승인 요청
- 생성된 결과 미리보기
- 다음 액션 버튼

이 패널은 사업 대시보드와 AI 실행 경험을 연결하는 핵심 영역입니다.

## 프로젝트 생성 흐름

사용자가 프로젝트를 만들 때 QARKO OS는 자연어로 사업 아이디어나 목표를 입력받습니다.

예시:

> 동네 식당을 대상으로 숏폼 마케팅 대행 사업을 시작하고 싶어.

QARKO OS는 이를 바탕으로 프로젝트 초안을 만듭니다.

- 프로젝트 이름
- 사업 목표
- 추천 단계
- 첫 작업
- 추천 AI 역할
- 리스크
- 추천 플러그인
- 초기 산출물 슬롯

프로젝트 생성 흐름에는 반드시 자동화/승인 범위 설정이 포함되어야 합니다.

## 자동화와 승인 범위

프로젝트 시작 시 사용자는 QARKO OS가 어디까지 자동화할 수 있는지 정합니다.

MVP는 네 가지 모드를 제공합니다.

### 수동 모드

AI는 리서치, 제안, 초안 작성은 할 수 있지만 의미 있는 실행은 모두 승인 후 진행합니다.

초보 사용자, 민감한 프로젝트, 익숙하지 않은 도구에 적합합니다.

### 보조 모드

AI는 리서치 요약, 개요, 체크리스트, 초안, 작업 정리 같은 안전한 내부 작업을 자동으로 수행할 수 있습니다.

외부 실행, 유료 도구 사용, 게시, 삭제, 플러그인 설치는 승인 필요입니다.

기본 추천 모드입니다.

### 자동화 모드

AI는 내부적이고 위험이 낮은 승인된 워크플로 단계를 자동 실행할 수 있습니다.

고위험 작업은 여전히 승인 필요입니다.

### 커스텀 모드

사용자가 작업 유형별 승인 규칙을 직접 설정합니다.

작업 유형:

- 리서치
- 초안 작성
- 파일 생성
- 이미지/영상 생성
- 외부 게시
- 유료 API 사용
- 플러그인 설치
- 데이터 삭제
- 프로덕션 설정
- 커뮤니티 공유

승인 카드는 항상 다음을 보여줍니다.

- 작업
- 실행 내용
- 예상 결과
- 위험
- 선택지: 승인 / 수정 / 취소

## AI 역할

MVP에서는 완전 자율 에이전트가 아니라 역할 카드로 표현합니다.

초기 역할:

- Chief of Staff: 모호한 목표를 우선순위, 마일스톤, 다음 액션으로 정리
- Product Manager: 요구사항과 사용자 흐름 정의
- Researcher: 도구, 경쟁사, 참고자료, 사실 조사
- Engineer: 기술 작업 계획과 구현
- Designer: UX, 레이아웃, 시각적 명확성 개선
- Growth Marketer: 콘텐츠, SEO, 유통, 수익화 계획
- QA / Reviewer: 품질, 리스크, 누락 상태, 승인 위반 점검
- Operator / Publisher: 승인 후 실행과 게시 준비

각 작업은 다음을 보여줄 수 있어야 합니다.

- 배정된 역할
- 상태
- 현재 단계
- 결과물
- 승인 필요 여부

## 플러그인과 커뮤니티

플러그인은 Codex Windows 앱의 플러그인 경험을 참고합니다.

MVP의 플러그인 갤러리에는 다음을 둡니다.

- 설치됨 탭
- 추천 탭
- 커뮤니티 탭
- 플러그인 카드
- 기능 요약
- 필요한 권한
- 관련 워크플로
- 연결 또는 설정 액션
- 프로젝트 호환성 표시

초기 샘플 플러그인:

- Hermes Agent Pack
- Browser Research Agent
- Blog SEO Assistant
- Shorts Workflow Kit
- Social Publisher
- Notion Sync
- Google Drive Artifacts
- Local File Operator

MVP에서 플러그인 동작은 대부분 mock 또는 로컬 상태 변경입니다. 목적은 제품 구조와 UI를 검증하는 것입니다.

향후 방향:

- 사용자가 워크플로를 템플릿으로 저장
- 사용자가 템플릿과 플러그인을 커뮤니티에 공유
- 커뮤니티 워크플로를 프로젝트에 설치
- 플러그인 활성화 전 권한과 위험 표시
- Hermes 에이전트/모델 조합을 프리셋으로 공유

## Hermes 연동

Hermes는 향후 에이전트 실행과 모델 선택을 담당하는 실행 제공자로 봅니다.

MVP는 깊은 Hermes 연동에 의존하지 않습니다. 대신 adapter 경계를 먼저 둡니다.

- `AgentRuntimeAdapter`
- `MockAgentRuntimeAdapter`
- `HermesAgentRuntimeAdapter`
- `ModelProviderAdapter`
- `PluginAdapter`
- `DesktopAdapter`

UI에는 Hermes/모델 상태를 실제 제품처럼 표시합니다.

- 연결됨
- 연결 안 됨
- Mock 모드
- 선택된 모델
- 사용 가능한 에이전트 팩

첫 구현은 mock 데이터를 사용하되, 이름과 인터페이스는 Hermes 연동을 염두에 둡니다.

## 추천 MVP 기술

사용자는 비개발자 창업자이므로, 첫 기술 선택은 빠른 제작, 쉬운 수정, 읽기 쉬운 구조를 우선합니다.

추천 스택:

- Vite
- React
- TypeScript
- Tailwind CSS
- lucide-react
- Zustand
- TypeScript 파일 기반 mock 데이터

향후 스택:

- Windows 데스크톱 앱 패키징: Tauri
- 로컬 데스크톱 저장소: SQLite
- 계정/동기화가 필요해질 경우: Supabase 또는 Postgres
- 실제 에이전트/모델 실행: Hermes adapter

첫 MVP에서는 Next.js보다 Vite를 추천합니다. 현재 우선순위는 서버 렌더링, 인증, 백엔드가 아니라 프론트엔드 제품 경험 검증이기 때문입니다.

## UX 원칙

QARKO OS는 다음처럼 느껴져야 합니다.

- 운영적
- 차분함
- 명확함
- 창업자 친화적
- 기술적이지 않지만 강력함
- 채팅 우선이 아니라 대시보드 우선
- AI가 무엇을 하는지 투명함
- 승인과 위험 작업에 안전함

피해야 할 것:

- 단순 채팅 UI
- 첫 화면을 마케팅 랜딩페이지로 만드는 것
- 과도한 장식 카드
- AI 작업을 모호한 로딩으로 숨기는 것
- 자동화가 통제 불가능하게 느껴지는 것
- 특정 사업 카테고리에 제품을 가두는 것

## MVP 성공 기준

MVP는 사용자가 다음을 할 수 있으면 성공입니다.

- QARKO OS를 열고 1인 사업 운영 대시보드라는 것을 이해
- 어떤 사업 아이디어든 프로젝트로 생성
- 프로젝트의 자동화/승인 범위 선택
- AI 역할 카드와 작업 카드 확인
- Codex 같은 실행 패널에서 진행 상황과 로그 확인
- 위험 작업 전 승인 요청 검토
- 워크플로가 만든 산출물 확인
- Codex식 플러그인에 가까운 플러그인 갤러리 확인
- MVP가 mock 실행이어도 Hermes 에이전트와 모델이 어떻게 붙을지 이해

사용자가 이런 느낌을 받아야 합니다.

> 이건 챗봇이 아니라 내 AI 회사 관제실이다.

## 구현 범위

첫 구현은 mock 데이터를 사용하는 완성도 높은 프론트엔드 MVP입니다.

우선순위:

- 레이아웃
- 내비게이션
- 대시보드 정보 구조
- 프로젝트 생성 흐름
- 자동화 정책 컨트롤
- 승인 카드
- 플러그인 갤러리 UX
- 실행 패널 UX

미루는 것:

- 실제 인증
- 실제 데이터베이스
- 실제 Hermes API 호출
- 실제 플러그인 설치
- 실제 게시
- 결제
- 커뮤니티 백엔드

## 구현 계획에서 정할 것

아래 결정은 구현 계획 단계에서 정합니다.

- 정확한 파일 구조
- Vite 새 프로젝트로 시작할지 여부
- 시각 테마
- mock 데이터를 TypeScript 객체로 둘지 작은 로컬 store로 둘지
- 첫 샘플 프로젝트와 워크플로

이 항목들은 MVP 설계를 막지 않습니다.
