# QARKO OS 작업실 중심 재설계

작성일: 2026-05-21

## 배경

QARKO OS의 목적은 Hermes를 터미널이나 PowerShell에서 직접 다루기 어려운 사용자가 Codex, Claude Code 같은 앱을 쓰듯 쉽게 AI 작업을 실행하도록 돕는 것이다. 현재 베타 버전은 Hermes 설치와 인증 터미널을 여는 수준에 머물러 있어, 실제 사용자가 “사업 아이디어를 입력하고 Hermes로 결과를 받는 경험”까지 자연스럽게 이어지지 않는다.

이번 재설계는 설정 앱이 아니라 작업 앱을 만드는 것이 목표다. 첫 실행에서 Hermes 설치, 모델 선택, 인증을 끝낸 뒤에는 사용자가 바로 작업실에서 프로젝트 목표를 입력하고, QARKO가 Hermes 실행, 승인 요청, 산출물 저장, 피드백 수집을 이어가야 한다.

## 참고한 제품 패턴

Claude Code 공식 문서는 Claude Code를 터미널 안에서 동작하는 agentic coding tool로 설명한다. 주요 패턴은 자연어 작업 지시, 코드/파일 작업, 명령 실행, 권한 제어, 설정 파일, hooks, subagent이다. 특히 `permissions.allow`, `permissions.ask`, `permissions.deny`처럼 도구 사용 권한을 명확히 나누고, 민감한 파일은 읽지 못하게 숨길 수 있다.

Codex 계열 도구 역시 작업 실행, 승인 모드, 로컬 파일 변경, 명령 실행, 변경 확인을 핵심 경험으로 둔다. QARKO OS는 이를 그대로 복제하지 않고, “코딩 전용” 대신 “1인 사업 운영 작업”에 맞춰 바꾼다.

## 제품 방향

추천 방향은 작업실 중심 앱이다.

QARKO OS는 사용자가 앱을 열었을 때 설정 마법사가 아니라 작업실을 먼저 만나야 한다. Hermes가 준비되지 않은 경우에만 첫 실행 체크리스트가 표시된다. 준비가 끝나면 사용자는 다음 흐름을 반복한다.

1. 오늘 진행할 사업 작업을 자연어로 입력한다.
2. QARKO가 작업을 작은 단계로 나눈다.
3. Hermes가 실제 모델로 작업을 실행한다.
4. 위험하거나 사용자 확인이 필요한 작업은 승인 대기 상태로 멈춘다.
5. 결과물, 로그, 변경사항, 피드백을 오른쪽 실시간 패널에서 확인한다.

## 샌드박스(안전 승인 모드)

샌드박스는 AI가 사용자 컴퓨터 전체를 마음대로 건드리지 못하게 만드는 안전한 작업 공간이다. QARKO OS에서는 이 개념을 사용자가 이해하기 쉽게 “샌드박스(안전 승인 모드)”라고 부른다.

사용자 설명 문구:

> 샌드박스(안전 승인 모드)는 AI가 내 컴퓨터 전체를 마음대로 조작하지 못하게 막는 안전 작업 공간입니다. QARKO OS는 프로젝트 폴더 안에서만 작업하고, 파일 생성/수정, 외부 명령 실행, 민감한 설정 변경은 사용자의 승인 후 진행합니다.

권한 모드는 세 단계로 제공한다.

1. 보기/초안 모드
   - 파일 변경 없이 계획, 글, 체크리스트, 문서 초안만 생성한다.
   - 초보자와 첫 실행 사용자에게 가장 안전하다.

2. 샌드박스(안전 승인 모드)
   - 기본값이다.
   - QARKO 프로젝트 폴더 안에서 작업한다.
   - 파일 생성/수정, 외부 실행, Hermes 명령, 네트워크 작업은 사용자 승인 후 진행한다.

3. 자동 실행 모드
   - 사용자가 명시적으로 켠 경우에만 사용한다.
   - 반복 작업을 자동 실행하되, 로그와 복구 가능한 기록을 남긴다.
   - 베타에서는 제한적으로만 제공한다.

## 첫 실행 흐름

첫 실행은 “Hermes 설정 마법사”가 아니라 “준비 체크리스트”로 보이게 한다.

체크리스트:

1. Hermes 설치 확인
2. 검증된 QARKO Hermes 버전 복구/업데이트
3. 모델 제공자 선택
4. 인증 방식 선택
5. 모델 선택
6. 테스트 작업 실행

중요한 원칙:

- 가능한 항목은 QARKO UI에서 직접 처리한다.
- Hermes가 반드시 대화형 터미널을 요구하는 경우에만 터미널을 열고, 사용자가 무엇을 해야 하는지 앱 안에서 안내한다.
- 터미널을 열더라도 QARKO는 다음 단계 상태를 계속 추적한다.
- 모든 문구는 한국어로 제공한다.

## 메인 작업실 UI

레이아웃은 Codex 데스크톱 앱과 유사한 실용적 구조를 따른다.

왼쪽 사이드바:

- 작업실
- 프로젝트
- Hermes 연결
- 플러그인
- 피드백
- 설정

중앙 작업실:

- 현재 프로젝트 목표
- 작업 입력창
- 모델/권한/자동화 범위 선택
- 실행 버튼
- 작업 단계 타임라인
- 승인 대기 항목
- 결과 요약

오른쪽 실시간 패널:

- Hermes 연결 상태
- 실행 로그
- 생성 산출물
- 변경사항 미리보기
- 승인 요청
- 화면 주석/피드백

오른쪽 패널은 항상 큰 3분할 화면이 아니라 Codex처럼 보조 패널로 작게 존재하고, 사용자가 로그/산출물/승인을 선택하면 해당 내용을 보여준다.

## Hermes 실행 모델

QARKO는 Hermes를 대체하지 않는다. Hermes를 사용자 대신 안전하게 호출하는 UX 레이어가 된다.

실행 원칙:

- 설치된 Hermes 실행 파일은 QARKO가 검증한 경우에만 사용한다.
- `hermes setup`, `hermes setup model`, `hermes auth add` 같은 대화형 작업은 필요한 경우 visible terminal로 열 수 있다.
- 일반 작업 실행은 앱 내부에서 hidden command로 실행하고 결과를 QARKO UI에 표시한다.
- 모델 제공자는 사용자가 선택한다.
- API 키는 기본적으로 장기 저장하지 않는다. 저장 기능이 필요해지면 Windows Credential Manager 또는 OS 보안 저장소를 사용한다.

## 베타 성공 기준

베타 테스터 3~5명이 다음을 할 수 있어야 한다.

1. 설치 파일로 QARKO OS를 설치한다.
2. QARKO 안에서 Hermes 설치/복구를 진행한다.
3. QARKO 안에서 모델 제공자와 모델을 선택한다.
4. OAuth 또는 API 키 방식으로 연결한다.
5. 사업 아이디어를 입력해 실제 Hermes 작업을 실행한다.
6. 실행 로그와 결과물을 확인한다.
7. 막힌 지점이나 UX 피드백을 QARKO에서 제출한다.
8. 피드백이 Discord로 전달된다.

## 다음 구현 범위

첫 번째 구현 범위는 베타 테스터가 막히지 않는 사용 흐름에 집중한다.

1. 설정 마법사를 한국어 준비 체크리스트로 전환한다.
2. 메인 화면을 작업실 중심으로 재배치한다.
3. 샌드박스(안전 승인 모드) 설명과 권한 선택 UI를 추가한다.
4. “오늘 할 작업” 입력 → Hermes 실행 → 로그/결과 표시 흐름을 강화한다.
5. 오른쪽 패널을 로그/산출물/승인/피드백 탭 구조로 정리한다.
6. 베타 피드백 제출 위치를 명확히 보여준다.

## 제외 범위

이번 범위에서는 다음을 하지 않는다.

- 자체 AI 모델 서버 구현
- Hermes 전체 기능 재구현
- 완전한 플러그인 마켓플레이스
- 결제/계정/팀 기능
- 모든 모델 제공자 자동 인증 완성

이번 목표는 상용화 전체가 아니라, 7~14일 베타 테스트에서 실제 사용 사례를 만들 수 있는 작업 가능한 버전이다.
