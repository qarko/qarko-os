# QARKO OS: Codex-style Hermes Workbench 설계

작성일: 2026-05-22

## 핵심 방향

QARKO OS는 사업 대시보드가 먼저 보이는 앱이 아니다. 기본 구조는 Codex Windows 앱과 같은 프로젝트 기반 AI 작업 앱이어야 한다. 차이는 Codex 대신 Hermes CLI를 실행 엔진으로 사용하고, 1인 사업자가 실제 사업을 만들고 운영하는 데 필요한 기능을 보조 레이어로 붙인다는 점이다.

따라서 메인 UX는 다음 한 문장으로 고정한다.

> 프로젝트를 열고, Hermes와 채팅하며, Hermes가 CLI로 실제 작업을 실행하고, QARKO OS가 로그, 파일 변경, 산출물, 승인, 사업 보조 기능을 정리해주는 Windows 작업 앱.

## 기준이 되는 Codex식 구조

QARKO OS가 먼저 따라야 할 구조는 다음이다.

1. 왼쪽 사이드바
   - 프로젝트 목록
   - 최근 세션
   - 새 프로젝트
   - 설정, 플러그인, 피드백

2. 중앙 작업 영역
   - 프로젝트 안의 채팅 세션
   - 사용자의 자연어 요청 입력
   - Hermes 응답
   - 실행 중 상태
   - 작업 결과 요약

3. 오른쪽 실시간 패널
   - 실행 로그
   - 명령 실행 상태
   - 파일 변경 목록
   - 산출물
   - 승인 요청
   - 웹/앱 미리보기
   - 화면 주석/피드백

4. 하단 또는 입력 영역
   - 메시지 입력
   - 모델 선택
   - 권한 모드 선택
   - 실행 버튼
   - 첨부/컨텍스트 추가

## QARKO가 Codex와 달라지는 부분

QARKO의 차별점은 메인 구조를 바꾸는 것이 아니라, Codex식 작업 흐름 안에 1인 사업 운영 기능을 자연스럽게 붙이는 것이다.

QARKO 전용 기능은 다음 위치에 둔다.

1. 프로젝트 생성 보조
   - 사업 아이디어를 프로젝트로 변환
   - 고객, 문제, MVP, 배포 목표를 초기 컨텍스트로 정리
   - 단, 사용자가 바로 빈 프로젝트를 만들 수도 있어야 한다.

2. 사업 실행 템플릿
   - MVP 만들기
   - Threads/Instagram 마케팅
   - 랜딩페이지
   - 피드백 수집
   - 고객 인터뷰
   - 상품화 체크리스트

3. 프로젝트 메모리
   - 사업 아이디어
   - 타깃 고객
   - 가격/상품/채널
   - 이전 Hermes 작업 결과

4. 피드백/커뮤니티
   - 베타 테스터 피드백
   - Discord 전송
   - 플러그인/스킬 공유

이 기능들은 메인 화면을 차지하지 않고, 사이드 패널, 프로젝트 설정, 템플릿 메뉴, 플러그인 형태로 제공한다.

## 첫 실행 흐름

첫 실행은 설정 마법사가 아니라 작업 앱 진입이어야 한다.

1. 앱을 열면 프로젝트 목록과 새 프로젝트 버튼이 보인다.
2. Hermes가 준비되어 있지 않으면 상단 또는 오른쪽 패널에 `Hermes 준비 필요`가 표시된다.
3. 사용자가 Hermes 설정을 누르면 준비 화면이 열린다.
4. Hermes 설치, 모델 제공자, OAuth/API key, 모델 선택을 진행한다.
5. 브라우저 인증 후에는 터미널을 기다리지 않고 `인증 완료 확인`을 누른다.
6. 연결되면 프로젝트 채팅으로 돌아온다.

중요한 원칙:

- Hermes 설정은 필수 게이트지만 제품의 중심 화면이 아니다.
- 사용자는 언제나 “프로젝트에서 Hermes와 대화한다”는 감각을 먼저 가져야 한다.

## 프로젝트 흐름

프로젝트는 QARKO OS의 기본 단위다.

프로젝트 생성 방식은 두 가지다.

1. 빈 프로젝트
   - 이름
   - 로컬 폴더
   - 권한 모드
   - 모델

2. 사업 프로젝트
   - 아이디어 입력
   - QARKO가 사업 컨텍스트 초안 생성
   - 고객/문제/MVP/채널을 프로젝트 메모리로 저장

프로젝트 안에서는 사용자가 Codex처럼 바로 채팅한다.

예:

- “이 폴더 구조를 읽고 MVP로 만들 첫 기능을 추천해줘.”
- “Threads 홍보용 랜딩페이지를 만들어줘.”
- “현재 오류를 찾아서 수정해줘.”
- “이 앱을 베타 테스터 5명에게 보여줄 수 있게 정리해줘.”

## Hermes 실행 모델

QARKO OS는 Hermes를 감싸는 UI 레이어다. 사용자는 터미널을 보지 않지만, 내부 실행은 Hermes CLI를 기준으로 한다.

기본 실행 흐름:

1. 사용자가 채팅 입력
2. QARKO가 프로젝트 컨텍스트와 권한 모드를 붙여 Hermes 실행 요청 생성
3. Hermes CLI 실행
4. stdout/stderr/log를 실시간 패널에 표시
5. 파일 변경/산출물/승인 요청을 분리 표시
6. 최종 응답을 채팅에 표시

Hermes setup/auth/model 기능은 별도 설정 화면에서 처리한다.

중요한 명령 계열:

- `hermes auth add <provider> --type oauth`
- `hermes auth status <provider>`
- `hermes auth list <provider>`
- `hermes model`
- `hermes doctor`
- `hermes chat -q ...`

## 권한과 샌드박스

사용자에게 보여줄 이름은 `샌드박스(안전 승인 모드)`다.

모드는 다음처럼 둔다.

1. 보기/초안
   - 파일 수정 없음
   - 계획, 문서, 초안 중심

2. 샌드박스(안전 승인 모드)
   - 기본 모드
   - 프로젝트 폴더 안에서만 작업
   - 위험 작업은 승인 대기

3. 자동 실행
   - 반복 작업 자동화
   - 게시, 삭제, 결제, 외부 전송은 승인 필요

4. 개발자 모드
   - 터미널, 브라우저, 하위 에이전트 등 고급 도구 허용
   - 베타에서는 명확한 경고와 함께 제공

## 오른쪽 패널

오른쪽 패널은 Codex처럼 작게 붙어 있는 실시간 작업 패널이어야 한다.

탭:

- 로그
- 변경 파일
- 산출물
- 승인
- 미리보기
- 피드백
- Hermes 상태

평소에는 좁은 아이콘 바이고, 탭을 누르면 패널이 열린다.

중요:

- 3분할 대시보드처럼 항상 넓게 차지하지 않는다.
- 사용자는 중앙 채팅에 집중한다.
- 오른쪽 패널은 “지금 Hermes가 무엇을 하고 있는지”를 확인하는 보조 화면이다.

## 현재 코드에서 바꿔야 할 것

현재 코드에는 사업 대시보드 중심 구조가 남아 있다. 다음 순서로 재배치한다.

1. `WorkspaceDashboard`
   - Codex식 채팅 워크벤치로 교체
   - 사업 카드/단계/우선순위는 제거하거나 보조 영역으로 이동

2. `ProjectView`
   - 프로젝트 상세 대시보드가 아니라 세션/파일/메모 설정 화면으로 축소

3. `ExecutionPanel`
   - 로그, 산출물, 승인 중심에서 파일 변경/미리보기/명령 상태를 추가
   - 오른쪽 보조 패널 역할 유지

4. Store
   - `activeRun.logs`를 채팅 메시지와 실행 로그로 분리하거나 명확히 타입 확장
   - 사용자 메시지, Hermes 응답, 실행 이벤트를 구분

5. Hermes Adapter
   - 데스크톱 앱에서는 실제 Hermes CLI
   - 웹 미리보기는 개발 편의용 mock
   - 단, 최종 베타 테스트 기준은 Windows 앱

## 베타 테스트 기준

테스터가 해야 하는 일:

1. 설치 파일로 QARKO OS 설치
2. Hermes 준비 화면에서 모델 연결
3. 새 프로젝트 생성
4. 프로젝트 안에서 Hermes와 채팅
5. Hermes가 실제 결과를 만들거나 작업 계획/파일 변경을 생성
6. 오른쪽 패널에서 로그/산출물 확인
7. 피드백 제출

성공 기준:

- 테스터가 터미널을 직접 열지 않고 Hermes를 사용할 수 있다.
- 프로젝트 안에서 “채팅으로 작업한다”는 느낌이 분명하다.
- QARKO의 사업 보조 기능은 유용하지만 메인 작업을 방해하지 않는다.
- 오류가 생기면 사용자가 어디서 막혔는지 설명할 수 있다.
